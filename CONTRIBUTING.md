# Contributing to the Wiki

Welcome! We’re building the definitive, verified resource for Derby Owners Club. We use a branching workflow to ensure our main repository remains stable.

## How to Contribute
1. **Check the Guidelines**: Read `GUIDELINES.md` first.
2. **Find a Task**: Check `TAXONOMY.md` for pages marked as "Stub" or "Needs Data."
3. **Pick a Template**: See `templates/Template_Library.md` to pick the right infobox.

## The Workflow
To protect the integrity of our project, please follow this process for all contributions:

1. **Fork the Repository**: Create a personal fork of this repository on GitHub.
2. **Clone your Fork**: Clone your fork to your local machine.
3. **Create a Feature Branch**: Never commit directly to `main`. Always create a new branch for your specific task:
```bash
   git checkout -b feature/your-feature-name```
4. **Draft & Submit**: Create your page in the relevant `content/` subfolder, push your branch to your fork, and open a Pull Request (PR) against this repository's `main` branch.

## Need Help?
If you are unsure how to use Git, reach out in the Discord thread. We are happy to help you get your environment set up!
